Replica

Functions​

OnClientEvent​

This item only works when running on the client. Client
</>
Replica:OnClientEvent(Callback: (...any) → nil) → {Disconnect: () → nil}

Listens for a client event, and fires the callback with the arguments passed.

Write​

This item only works when running on the server. Server
</>
Replica:Write(
Method: string,
...: any
) → any

Uses the WriteLib to write data changes, and then replicates the write call to the client along with any information needed to replicate the call.

SetValue​

This item only works when running on the server. Server
</>
Replica:SetValue(
path: {string},
value: any
) → ()

Updates the specified path to the provided value. -- This isn't the recommended use of the API, and should only be used in cases of backwards compatibility. -- Credit to @Fuse for the backwards compatibility implementation.

GetChildren​

</>
Replica:GetChildren() → {[string]: Replica}

Returns all children of the replica, if the replica is not a parent it will return an empty table.

OnServerEvent​

This item only works when running on the server. Server
</>
Replica:OnServerEvent(Callback: (
Player: Player,
...any
) → nil) → {Disconnect: () → nil}

Connects a function to the server event, which will be fired when the client sends a request.

Fire​

</>
Replica:Fire(...: any) → nil

Sends a request to the client, sending the arguments to the client.

SetParent​

This item only works when running on the server. Server
</>
Replica:SetParent(Parent: ServerReplica) → nil

Sets the parent of the replica, and then replicates the change to the client.

ReplicateFor​

This item only works when running on the server. Server
</>
Replica:ReplicateFor(Player: Player) → nil

Replicates the replica to the specified player. This should only ever be intentionally called from the Server. -- Credit to @Fuse for implementation of this function.

Destroy​

This item only works when running on the server. Server
</>
Replica:Destroy() → nil

Destroys the replica and cleans up any tasks that were added to the cleaner. [Destroying] will be fired when the replica is destroyed. This should only ever be intentionally called from the Server.

DestroyFor​

This item only works when running on the server. Server
</>
Replica:DestroyFor() → nil

Destroys the replica for the specified player. This should only ever be intentionally called from the Server. -- Credit to @Fuse for implementation of this function.

ListenToWrite​

</>
Replica:ListenToWrite(
Method: string,
Callback: (...any) → nil
) → RBXScriptConnection

Connects to the Write event and listens for a specific method to be called. When the method is called, the callback is fired with the arguments passed to the method.

ListenToKeyRemoved​

</>
Replica:ListenToKeyRemoved(
Path: {string | number} | string,
Callback: (
Key: string | number,
Value: any
) → nil
) → RBXScriptConnection

Connects to the Change event and listens for a specific key to be removed. When the key is removed, the callback is fired with the key and the value of the key before it was removed.

ListenToNewKey​

</>
Replica:ListenToNewKey(
Path: {string | number} | string,
Callback: (
Key: string | number,
Value: any
) → nil
) → RBXScriptConnection

Connects to the Change event and listens for a new key to be added. When the key is added, the callback is fired with the key and the value of the key.

ListenToChange​

</>
Replica:ListenToChange(
Path: {string | number} | string,
Callback: (
NewValue: any,
OldValue: any
) → nil,
SuppressOld: boolean?
) → RBXScriptConnection

Connects to the Change event and listens for a specific key to be changed. When the key is changed, the callback is fired with the new value and the old value.

ListenToRaw​

</>
Replica:ListenToRaw(Callback: (
Path: {string | number},
NewValue: any,
OldValue: any
) → nil) → RBXScriptConnection

Connects to the Change event and listens for any key to be changed. When a key is changed, the callback is fired with the path, the new value, and the old value.

ObserveChange​

</>
Replica:ObserveChange(
Path: {string | number} | string,
Callback: (Value: any) → nil
) → RBXScriptConnection

Observes a specific key and fires the callback with the value of the key. The callback is fired immediately and then whenever the key is changed.

AddCleanupTask​

</>
Replica:AddCleanupTask(
Task: () → nil,
CleanupMethod: string | boolean?
) → nil

Adds a task to be cleaned when the replica is destroyed. If a method is provided it'll use this method to clean, otherwise it'll assume that it is a function or thread (true).
